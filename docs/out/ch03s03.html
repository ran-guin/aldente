3. Editing Records

To make changes to entries, you may utilize any of the hyperlinks that point to database info.

From most of these pages there are options below to Edit the entry, (or search the database for another entry)

There are also more direct routes to search/edit provided via the 'Plates', 'Libraries' and 'Sample Sheets' pages for editing the Plate, Library or Sequence (Run info) records respectively.

When making changes, note the following:

  • use 'Save Changes' to CHANGE current values.

  • Use 'Save Changes as New Record' to add a NEW record.

Note when Saving changes as a new record:

  • Sometimes UNIQUE fields are shown (eg. Run ID). In these cases, the value must be erased (ID's will automatically generated), or changed (such as Sequence_Subdirectory which must also be unique)

3.1. Annotating records with notes

Sometimes mistakes are made during laboratory procedures that cause errant records to be placed in the database. These can often be removed from the database.

Removing invalid plates: Plates can be removed by going to the 'Plates' page icon and then selecting the Check Recent Plates Button. This will bring up plates made by the same user within the last 7 days.

From here users can select any number of plates for deletion (you may instead mark plates as failed, or annotate any number of plates with a common note)

Note: If any other data points to a plate, you will NOT be able to delete it. (if for example a sequence run uses a plate - then you will not be able to delete that plate).

Removing Sequence Requests (sample sheets): Requests for Sequence Runs can be erased in a similar fashion by going to the 'Sample Sheets' page and selecting 'Remove Sequence Requests'. Again the user will be given a list of all requests that they have made that have not yet been analyzed.

Removing Solutions (or Reagents): Solutions can be deleted in a similar fashion. By clicking on 'Check my Recent Solutions', the user may select any number of recently made solutions for deletion (as long as they have not been used in the meantime). The same page allows users to reprint a number of barcodes at the same time.
