2. Sequence Runs

2.1. Lab Tracking

2.1.1. Requesting

In order to request a Plate to be sequenced you must scan the given Plate and the Sequencer that will be used to sequence it, in the Scan Field and click on “Scan”. As shown in the figure below, the system displays some specifics about the scanned Plate and Sequencer and prompts for:

Figure 14.1. Sample Sheet Generation Form

  • Select the Run Test Status

  • Quadrants to Sequence

  • Enter Run Batch Comments

  • Some Sequence Run specific options such as

    • Protocol

    • Sealing

    • Terminator

    • And some Sequencer specific options such as Run Time, Run Voltage, etc.

You can create the Sample Sheet by clicking on the “Generate Sample Sheet” button.

Before creating a sample sheet you must make sure:

  1. You have selected the Run Test Status flag as either Production or Test

  2. The Plate is Active and has have a Primer applied to it previously

  3. The Sequencer contains valid Buffer and Matrix solutions

If no errors occur during the process of Sample Sheet generation, you can be found them in /home/sequence/Projects/[Project_Name]/[Library_Name]/SampleSheets/. At this point the Sequence Runs have been created in the database and their Run Status has been set to “In Process”.

Figure 14.2. Generated Sample Sheets

You can now load the generated Sample Sheet on the Sequencer and the Plate is ready to be sequenced.

2.1.2. Monitoring

In the generation of high throughput sequencing information, it is essential to have constant feedback as to the lab-wide status of various samples, allowing lab administrators to identify pending processes, bottlenecks, or data requiring closer observation.

After the Runs have been started (by means of having their Sample Sheets generated) you can view them in the Last 24 Hours Page by clicking on the “Last 24h” link on top of the screen under “Sequencing” tab. These Runs will appear with Run Status of "In Process."

The Last 24 Hours Page. This page is called Last 24 Hours since by default it will show all the Runs that have been requested within the previous day, but you can change this time frame and view Runs within any time frame, as well as searching by:

  • Project

  • Sequencer Machine

  • Primer Used

  • Range of Run IDs

  • Library Name of the Plates

  • Run Status

  • Run Test Status

  • Validation Status

  • Plate Class

Some of other functionalities that this page provides are:

  • Monitoring of the status of samples within a specified library. This provides a quick view of which samples have yet to be sequenced, which chemistries have been performed, and the number of samples at each position within the protocol.

  • Display of the Run Validation flag, Plate Class (St: Standard, Re: ReArray, and Ol: Oligo)

  • Display of whether the Plates were reloaded or not beside the Run Maps. “R(2/2)” indicates that this is the second of two plates of reloaded plates. “R+(2/2)” indicates that this plate was reloaded as another plate. (Possibly re-prepped)

  • Quick views of recent runs, displaying average quality lengths, vector components, chemistry information as well as an easy to view colour-coded map indicating quality distribution of reads within a plate.

  • Quick views summarizing the results by library or project.

  • Automatic Diagnostics checking, which correlates sequence quality with numerous subtle variables within the lab setting which could have an effect on final data quality. These correlations involve equipment, solutions, and processes applied to samples prior to sequencing, as well as more direct factors such as which sequencing machine was used.

  • Complex Database summaries may also be generated, with a whole suite of highly informative visual reports that represent data in a variety of formats.

2.1.3. Analysis

After the Runs have been completed and raw data is stored on local Sequencer computer. These data are mirrored regularly (on request, as well as daily via a cron job) onto /home/sequence/mirror. This entails copying all of the Data Trace files. At this point Update Sequence script which is running as a cron job as well will do the following:

  1. Generate the appropriate directories in /home/sequence/Projects/[Project.Project_Name]/[Library.Library_Name]/AnalyzedData/

  2. Creates symbolic links to all of the chromat directories in the mirror directory (placing them in chromat_dir/)

  3. Runs 'phred' on the chromat_dir (This results in a number of statistical files which include phred scores for each nucleatide generated), placing the resultant files in the directory phd_dir

  4. Extracts information from these “phred” files and places the appropriate information in the Clone_Sequence table (as well as some subsidiary information such as the well Growth Status ('OK','No Grow' or 'Slow Grow'), Warnings (poor quality, lack of data, vector only, recurring string etc.)

  5. Generates a colour map gif file of the 96-well plate (done separately for each of the sub-plates of a 384-well plate) - and placing a copy in /www/htdocs/intranet/SDB/RunMaps/

  6. Sets the Run_Status of the Runs to “Analyzed”

  7. Deletes any current information on this run (in case it was analyzed before)

  8. Updates information in the database on this read.

2.1.4. Validating, Failing and Commenting Runs

From the Last 24 Hours page the user can select multiple Runs and

  • Set their Run Validation flag to

    • Approved

    • Rejected

    • Pending

  • Fail them by selecting a valid Fail Reason.

  • Add Comments to each individual Run

Note

When Failing a Run or changing the Run Validation flag to “Rejected”, specifying a comment is mandatory.

2.2. Tracked Data

These are type of runs when a plate which contains samples and a primer has been loaded on a Sequencer. Details recorded for each Sequence Run are as follows:

  • Chemistry Code Name. 

  • Primer Solution. Primer Solution that was previously applied to the Plate which this Run has been initiated from

  • Buffer/Matrix Solutions. Buffer and Matrix Solutions that were on the Sequencer at the time of this Run

  • DNA Volume. 

  • Total Prep Volume. Primer Solution that was previously applied to the Plate which this Run has been initiated from

  • Brew Mix Concentration. Primer Solution that was previously applied to the Plate which this Run has been initiated from

  • Reaction Volume. 

  • Resuspension volume. 

  • Slices. 

  • Run Format. 96-well, 384-well or Tray plate

  • Run Module, Time, Voltage, Temperature. 

  • Injection Time, Volume. 

  • Plate Sealing. Septa, Foil or Heat

If a Sequence Run has been analyzed, the following information will be stored related to its analysis

  • Analysis Date Time. Date and Time for when the raw data has been analyzed

  • Phred Version. 

  • Q20: The Array, MIN, MAX, Mean, Median, Total. 

  • SL: The Array, MIN, MAX, Mean, Median, Total. 

  • QV: Mean, Total. 

  • QL: Mean Total. 

  • Wells, No Grows, Slow Grows. 

  • All Reads, Successful Reads, Trimmed Successful Reads. 

  • All BPs. 

  • Vector Segments Warnings. 

  • Vector Only Warnings. 

  • Contamination Warnings. 

  • Recurring String Warnings. 

  • Poor Quality Warnings. 

  • SS Totals (A,G,C,T).
