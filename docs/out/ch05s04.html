4. What is an Attribute ?

An attribute is a dynamic field that can be defined and set by the user. Attributes can be defined for various objects already in LIMS such as Sources, Libraries, Plates etc.

4.1. How to define an attribute (Administrators only)?

  • Locate the “Attributes: Add new or Set existing” link on the homepage of the object for which you want to create the attribute and click on “Add new”.

  • Fill in the Attribute name, the group to which the attribut applies and wheather or not it is inherited.

    Note

    When an attribute is inherited it means that the value of that attribute is passed onto the child objects. In case of plates the attribute of a particular plate will be passed on to all daughter plates that are derived from the current (parent) plate.

  • Cick the “Update Attribute” button to complete the action.

4.2. How to set the value of an attribute?

Important

In order to set the value of an attribute it must must be previously defined (see 4.1)

  • Locate the "Attributes: Add new or Set existing" link on the homepage of hte object for which you want to set the attribute value and click on “Set existing”

  • Select the Attribute you wish to set, enter the value for the selected attribute and click the “Update Object_Attribute button”.

4.3. How to view attribute values?

To view the values of attributes set for a particular object simply go to the objects homepage by scanning the object's barcode or navigating to the hompage by other means. The attribute names and their values will be displayed in the record details table on the right hand side of the page.

4.4. How to edit attribute details?

To edit the attribute values simple click on the attiribute value you want to edit, change the value as desired and save the record.

4.5. Attributes in protocols.

Attributes can also be set for protocols steps. Each protocol step can be associated with two different kinds of attributes:

Table 5.1. Attribute Classes

Prep AttributesPrep level attributes apply to all plates/tubes that go through the protocol step for which the prep attribute is recorded eg. Thaw temperature, Thaw time, etc
Plate AttributesPlate level attributes apply to each individual plate allowing user to enter a different attribute value for each plate in the protocol step by separating the attributes values with semicolons eg. Concentration, OD, etc

4.5.1. Adding Prep Attributes (Administrators only).

To add a prep step to a protocol step:

  • Navigate to the desired step from the protocol edit page.

  • In the Input section of the Step details select the checkbox on the row labeled “Prep Attribute”

  • Select the desired Prep Attribute from the dropdown menu and give it default value (optional)

    Note

    If the Attribute you want to add does not exist in the dropdown list click the “Define new” link and fill in the new attribute details as in section 4.1 above.

  • Multiple Prep Attributes can be added to the same protocol step by clicking the “+” button in the Options column of the Input section.

    Clicking the “+” button simply duplicates the current table row which can then be used to add another prep attribute to the step.

  • In order to remove unwanted Prep Attributes simply click the “-” button corresponding to the row of that attribute and it will be removed.

4.5.2. Adding Plate Attributes (Administrators only).

To add a Plate Attribute to a protocol step simply follow the steps described above (Sec4.5.1) for adding Prep Attributes selecting the Plate Attribute instead of the Prep Attribute.
