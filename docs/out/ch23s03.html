3. alDente Modules

This directory includes modules specifically designed for use by the alDente LIMS. Modules within this directory also require those in the 'SDB' directory (also included in the package), and modules in the 'RGTools' directory (also included in the alDente package). Additional standard modules required include: GD::Graph

  • Button_Options

    Module that determines which modules/functions are used given a set of form parameters.

  • Special_Branches

    Custom code that gets triggered after Button_Options finishes.

  • RNA_Source

  • Help

  • Tube

    This module handles Container (Plate) based functions

  • Sequence

    This module handles routines specific to Sequencing (Sample Sheets set up)

  • Transposon_Pool

    Transposon pooling

  • ReArray

    This modules handles functions related to rearraying

  • Seq_Data

  • Library_Plate_Set

    This module handles Plate-based Set functions

  • Primer

    Module that handles primer inserts.

  • RNA_Original_Source

  • Contact

  • Validation

    Module that contains database integrity validation code

  • Clone_Sample

    This module handles Clone_Sample based functions

  • SDB_Status

    This modules provides various status feedback for the Sequencing Database (such as the Last 24 Hours page)

  • Phred (deprecated)

    Module for running phred outside of Post.pm

  • Comments

    Module that allows comments to be posted to admininstrators

  • Read

    This module is used to handle 'Read' objects

  • Collaboration

  • Admin

    This modules provides various functions for Administrative Purposes

  • Library_Plate

    This module handles Container (Plate) based functions

  • Collaborator

  • Submission

  • Sample_Sheet

    This module handles routines specific to Sample Sheet generation for sequencing

  • Employee

    This module is used to handle Employee objects and functionality

  • Seq_validation

  • Process

    This facilitates protocol processing

  • Sample_Pool

  • Run_Statistics

    Module that contains code for generating summaries and reports.

  • Run

    This module handles routines specific to Sequencing Runs

  • Clone

    This module handles Clone based functions

  • Prep

    This module is used to handle preparation tracking of Plates/Tubes etc.

  • Library_Pool

  • Stock

    This module handles receiving new stock (reagents/equipment).

  • Study

    Encapsulates a Study

  • Misc_Item

    This module handles Miscellaneous Items.

  • Box

    This module handle Box stock items.

  • Plate_Prep

    This module is used to handle Plate_preparation tracking of Plates/Tubes in conjuction with Prep.pm

  • Notification

    This modules is a wrapper around sendmail for sending email messages.

  • Sample

    This module handles Sample based functions

  • Summary

  • Library

    This module contains library creation functions (internal and external/submission), as well has homepage generation and accessors.

  • Sequencing_Library

    Subclass of Library.pm. Handles sequencing library-specific functions.

  • DBIntegrity

    Module that handles foreign key integrity checks

  • Security

  • Container

  • Chemistry

  • Department

  • Extraction_Sample

    This module handles Extraction_Sample based functions

  • Pool

    Standard pooling module

  • Container_Set

    This module handles Container Set based functions

  • Protocol

  • Vector

  • Rack

  • This module supplies some basic Sequencing database diagnostics checks.

  • Messaging

  • Fasta

  • Post

    Module that handles chromatogram analysis

  • Info

    This modules provides general info retrieving functions specific to the SeqDB

  • Well

  • Orders

  • Equipment
