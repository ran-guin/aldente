3. Created 384-well plate instead of 96-well plate by mistake

If nothing was done with the plate, simply delete it, and create again. Otherwise, slightly more work is required.

  • check ancestry of given plate and start correcting at the end (ie newest plates in ancestry)

  • delete all aliquoted plates belonging to sub-quadrants that do not exist (b-d), and clear Plate_Position, Parent_Quadrant fields in 96-well plates aliquoted out.

  • edit plate_format as required for all daughter 384-well plates except original.

  • create new correct original plate of proper format.

  • Correct Plate_Created, Plate_Created fields in new plate to reflect original values

  • UPDATE Plate_Prep set FK_Plate__ID=<new_id> where FK_Plate__ID = <original_id>

  • UPDATE Plate set FKParent_Plate__ID = <new_id> where FKParent_Plate__ID = <original_id>

  • delete original plate via the interface. (to ensure no other cross-references pending) - this will cascade delete the Sample and Plate_Sample references.
