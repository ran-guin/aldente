2. Standard APIs

Documents may be retrieved via the LIMS by simply typing in 'API' in the help section and clicking on the appropriate links.

The current API is not a true API in that it is not independent from the production code. It has been set up to ensure that all queries to the database (and updates) from outside the LIMS itself goes through a standard protocol. In this role, it provides an effective method of insulating outside users from internal changes to database structure. By filtering outside calls through the relatively narrow functionality within this one module, it also allows the LIMS team to maintain this portion of the code with added rigour without putting an excessive strain on our resources.

The intention is to slowly build up the flexibility and robustness of the API to enable it to perform as a true API and support web services down the road, but in the meantime, the priority is to perform this limited but valuable role.

Documentation for the various API's is generated automatically directly from the code. If there are any sections which are poorly commented, please let us know so that we can upgrade them.

2.1. alDente_API

The standard alDente_API provides access to generic information within the LIMS which is not department or application specific.

Both the perldoc and information for the various methods can be accessed at: http://seqdb01/SDB/html/perldoc/alDente_API.index.html

The perldoc itself is at: http://seqdb01/SDB/html/perldoc/alDente_API.html

2.2. Sequencing_API

The Sequencing API provides access to database information that is specific to the Sequencing group and includes such methods as 'get_read_data' or 'get_SAGE_data'.

Both the perldoc and information for the various methods can be accessed at: http://seqdb01/SDB/html/perldoc/Sequencing_API.index.html

The perldoc itself is at: http://seqdb01/SDB/html/perldoc/Sequencing_API.html
