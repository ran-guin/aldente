1. Definition

A Run can be any sort of experiment in which samples are processed on an Equipment by an employee in order to generate raw data from the samples for further analysis.Runs are usually started in Batches of one or more Runs. For each batch LIMS records Who (Employee that requested the batch), When (time when Batch was requested), and Where (the equipment that was used for this Run). These parameters are the same for all the Runs in a given Batch.

Primary details recorded for Run Batches

  • Run Request Date Time. The date and time when a Batch of Runs were requested. For eg: when a Sample Sheet was generated

  • Employee. The LIMS Employee who ordered the request for these Runs

  • Equipment. The equipment that will generate the raw data for these Runs. For eg: Sequencer or Flouroimager

  • Comments. Any comments regarding this Batch of Runs

Primary details recorded for Runs

  • Run ID. The unique identification number for the run

  • Plate ID. The Plate that is associated with this Run

  • Run Type. Sequence Run, Gel Run or Genechip Run

  • Run Date Time. The date and time when the Run was successfully finished and raw data were imported into the system

  • Run Test Status. Production or Test

  • Run Status. In Process, Analyzed, Aborted, Failed, Expired or N/A

  • Run Directory. The directory in the file system where the raw data from this run is stored

  • Run Validation. Pending, Approved or Rejected

  • Billable. Yes or No

  • Comments. Any comments related to this Run. For eg. Reason why this Run was rejected

  • Excluded Wells. Wells for which the obtained run data will not be stored or analyzed.

All the information recorded for a Batch will be inherited to all the Runs in that Batch. LIMS Currently supports two types of Runs, Sequence Run and Gel Run.

Note

LIMS supports the tracking of Attributes for Runs. This will allow for tracking other parameters dynamically as the needs arise.
