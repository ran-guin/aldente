1. QC_Status as a Plate_Attribute

  • Enable solutions or plates to be “flagged ”as passed or failed, or set to be re-tested

  • Passed plates or solutions can be identified and located in the LIMS

  • Protocols can stop a plate or solution if the QC status has been set and the status has not been set to passed

1.1. QC Status

Plates and solutions now have an extra status for tracking their quality control.

The QC status field will contain the following values:

  • N/A

    • By default, the QC status is set to N/A (plates can be processed even though they have not been taken through a QC process. The system does not require users to use passed plates or solutions.

  • Pending

    • Plates and solutions are initially set to a QC status of Pending when the QC process is started

  • Failed

    • A plate or solution undergoing QC can be set to a status of failed; the plate can then be thrown out at this point

  • Re-Test

    • A plate or solution that has failed a QC test for whatever reason may be set to a status of Re-test, meaning it can be tested again

  • Passed

    • Once an item has passed QC, it has a status of Passed

1.2. Roles in Protocols

Currently if the QC status of an item exists, the system will check whether or not the item has passed QC.

Requirements for Plates and Solutions used in Protocols

  • Plates that have been marked as having a QC status other than passed, will not be allowed to enter a protocol.

  • Solutions that have been marked as having a QC status other than passed cannot be applied to a plate

1.3. Beginning the QC Process

The user can start the QC process by scanning in the item(s) to be processed

To begin the QC process for a plate or solution:

  1. Scan in the plate(s)/solution(s).

  2. Click on the Begin QC button

  3. The QC Status for the plates are set to pending

1.4. Updating the QC status

The user can update the status of items that have a status of Pending or Re-Test. The available choices would be to Pass or Fail the item

To update the QC status for a plate or solution from a scanner:

  1. Scan in the items that have a QC Status of Pending or Re-Test.

  2. Choose to pass or fail the item

  3. Click on Update QC Status

1.5. Failed QC status

The plate or solution will be thrown out when the QC Status is set to Failed

1.6. Passed QC status

The item can be used in protocols.

1.7. Locating Items based on QC status

The user can view a list of solutions or plates and their location based on the QC status. For example, the user can view a list from the plate main page finding all the plates that have been failed or pending.
