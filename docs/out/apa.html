Appendix A. Terms and Definitions

  • Trace Data

    The raw trace Data for sequences is stored in an archive directory. Within each Project/Library directory there are also directories containing traces. These directories contain links to the archive directory.

    Storage Structure. 

    • Mirror: A temporary repository for files as they are copied over from the sequencers.

    • Archive: A storage directory containing all trace files (organized by sequencer) eg: 3700/2/data1/data

    • Within each machines data directory there are a number of directories - each containing a set of read data (usually 96 trace files). In addition there is a link created pointing to this directory which bears the name of the 'Sequence_Subdirectory' (link). This allows traces from specific libraries or plates to be easily found regardless of the naming convention of the original directories created.

    • Projects: A central directory for storing information by project. Trace data in the project directory may be found by following the specified directory structure. eg: /Forestry/WS001/AnalyzedData/WS0011a.B7/chromat_dir/. This naming convention is unambiguous and the final directory contains links to the trace data for this run in the archive directory.

    Viewing Trace Data. Generally Trace data is in the form of '*.abd' files (ABI 3700 or 3730 sequencers), or '*.ab1' files (Megabace sequencers).<P> One simple way to view a trace is to use the command line function 'quick_view'. This extracts information for the read as well as opening up a trace window (using 'ted' - a widely used and simple chromatogram viewer). The syntax for using this command is explained if you simply type 'quick_view' on the command line without any options.
