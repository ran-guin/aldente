3. Automated Testing

To increase the efficiency of testing, the LIMS system has a built in testing engine, which enables a number of methods to be automatically tested every night by comparing the output from the development code & development database to the same call using the current production code / production database. This quickly and easily identifies any discrepencies which result in different output being provided by the method. This is particularly useful for the API, enabling automated consistency checking on a regular basis with minimal overhead.

To enable debugging as well as consistency checking for the API, the LIMS logs API calls, storing standard login parameters (user, database, host etc) as well as the input parameters to the API method. The test engine then generates the output for this call (this is truncated to two records to avoid logging the output for queries that generate large chunks of output data). This output is appended to the log to enable the regular testing engine to compare the results with the results obtained by using the development code / database. Whenever there is a discrepency between the outputs, the LIMS developers are notified and provided with a clear summary of the discrepency, allowing them to quickly and easily address unforeseen bugs that may arise.
