2. Submission Process

This chapter describes the process of submission through LIMS. The submissions will get validated and approved before loaded to the database. Email notifications will be sent to users detailing the fate of their submission.

Note

The batch submission feature is under development.

2.1. Submission Data Types

Submissions can either contain one single object and all its entities, or can be a Batch Submission in which case contains one ore more objects and they all get entered into the database at once.

  • Single Object Submissions

    Single Object Submission are resulted from the user going through and filling out different forms within LIMS while they do not have permission to directly record that information into the system.

  • Batch Submissions

    Batch Submission are used when a user or collaborator submits a single Comma Separated Value (CSV) file. This CSV file should be obtained as a template from LIMS and validated before entered into the database

    Note

    Batch Submission is still under development.

2.2. Submitting User

Submitter of a submission has to be either a GSC Employee or a GSC Contact. If the submission is done through alDente LIMS External Collaborator page, the submitter will be the Contact ID of that collaborator, otherwise it will be the Employee ID of that employee

2.3. Submission Groups

There are always two groups associated with a submission, a TO group and a FROM group.

  • To Groups

    To Groups are Groups which have Guest access in LIMS

  • From Groups

    From Groups can be the users's current group, or in case of External submissions is set as the External group

2.4. Submission Status

There are six statuses associated with Submissions. They are Draft, Submitted, Approved, Completed, Cancelled, Rejected

  • Draft Submissions

    These are submissions that are created by any user, but are not fully submitted to the "To" group. Submitter can retrieve this submission, Edit it, Cancel it or Submit it.

  • Submitted Submissions

    These are submission that are submitted to the LIMS system and are pending approval

  • Approved Submissions

    These are submissions that have been approved by an administrator of the "To" group, the data has been stored in the database. Once submissions are approved, the "Key_Value" field of the submission gets set to the primary ID of that submission object. Except in case of Batch Submissions. Key_Value together with Table_Name can uniquely identify the main object of a submission

  • Completed Submissions

    These are submissions that have been approved and are archived.

  • Cancelled Submissions

    These are submissions that were Drafted, but the user has decided not to submit them any more.

  • Rejected Submissions

    These are submissions that were rejected by the administrator of the "To" group. The submitter either has to inquire about this rejection and it might be needed to be resubmitted.

2.5. Submission Approval Process

Any administrator in which a submission is submitted to his/her group can approve/reject/edit a submission. Once a submission has been approved, an email will be sent out to the Administrators of the To and From groups, as well as the submitter.
